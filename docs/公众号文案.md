## 正文

### 一、写简历的 N 个崩溃瞬间

如果你正在求职，下面这些场景你一定经历过：

**排版地狱。** 用 Word 写简历，字号对不齐、行距忽大忽小、加个项目符号整页跑偏。改了两小时排版，内容一个字没动。

**模板焦虑。** 在线简历网站模板倒是好看，导出 PDF 要开会员，换个颜色又要加钱。免费的要么带水印，要么只有英文模板。

**不会写成果。** 通篇"负责后端 API 的开发与维护"、"参与了 XX 项目"，自己看着都觉得没说服力，但又不知道怎么改。

**一份走天下。** 投大厂和投银行用同一份简历，投前端岗和投算法岗也是同一份。石沉大海，不知道哪里出了问题。

**AI 帮了一半。** 用 ChatGPT / DeepSeek 润色了内容，但输出是一堆纯文字，排版还得自己搞。等于只解决了一半问题。

这些问题我全遇到过。找了一圈工具都不满意，所以自己做了一个。

### 二、resume-builder：从内容到 PDF 成品，一步到位

resume-builder 是一个 Claude Code / Codex Skill。

它不是一个 prompt 模板，也不是一个在线网站——它是一个完整的简历生产工具链：

> **对话采集经历 → AI 润色内容 → 选模板选配色 → 一行命令渲染 A4 PDF**

整个流程在本地完成，不上传数据，不需要网络，不要会员。输出的 PDF 文字可复制、可搜索，直接投递。

用一句话调用就行：

```
使用 $resume-builder，根据我的经历生成一份后端校招简历
```

### 三、五个核心能力

#### 1. STAR 句式润色：把流水账变成有说服力的成果

大部分人写简历的习惯是列职责——"负责了什么"、"参与了什么"。但招聘方想看的是成果，不是工作日志。

resume-builder 会用 STAR（情境→任务→行动→结果）和 XYZ（通过做 Z，达成 X，量化为 Y）句式改写你的每条经历，并且追问你要数字——规模多大、提升了多少、节省了多少时间。

**改写前：**
> 负责后端 API 的开发与维护。

**改写后：**
> 优化任务调度服务，引入本地缓存与批量写入，将 P99 延迟从 640ms 降至 210ms，支撑日均 920 万次请求。

这不是 AI 编的数字——是从你的真实经历里挖出来的。AI 的作用是帮你组织语言和找到值得量化的角度，而不是帮你造假。

#### 2. 五套专业模板，一行命令切换

同一份简历内容（Markdown），可以随时切换模板重新渲染：

| 模板 | 风格 | 最适合 |
|------|------|--------|
| `compact` | 紧凑单栏、清晰稳定 | 中文技术岗、校招、内容较多想压一页 |
| `classic` | 黑白工整、ATS 友好 | 海投、国企、银行、保守行业 |
| `modern` | 侧栏 + 彩色标题 | 互联网、产品、运营岗位 |
| `timeline` | 时间线布局 | 实习/项目经历多、想强调成长脉络 |
| `minimal` | 极简大留白 | 内容精炼、偏设计感 |

不知道选哪个？**技术岗选 compact，拿不准选 classic，想出彩选 modern。**

国内求职需要头像时，也可以在 Markdown 中增加 `photo` 字段。`compact`、`classic`、
`modern` 和 `minimal` 会自动完成裁切与排版；不需要照片时删除该字段即可，`timeline` 默认无照片。

#### 3. 七种配色 + 自定义色值

预设了 7 种配色：蓝、青绿、酒红、墨黑、紫、绿、焦橙。也可以直接传 `#rrggbb` 色值。

同一个 modern 模板，换成酒红和换成青绿是完全不同的气质。classic 模板始终保持黑白，保证 ATS 系统能正确解析。

```bash
python3 resume_skill/scripts/render.py resume.md --template modern --accent wine --out resume.pdf
python3 resume_skill/scripts/render.py resume.md --template modern --accent teal --out resume.pdf
python3 resume_skill/scripts/render.py resume.md --template compact --accent blue --out resume.pdf
```

一行命令换皮，不用重新排版。

#### 4. JD 匹配分析：从海投变成精准投递

给 resume-builder 一份目标岗位的 JD，它会自动：

1. **抽取关键词**：从 JD 中提取硬技能（语言/框架/工具）和软要求（owner 意识、跨团队协作等）
2. **匹配度分析**：告诉你简历命中了哪些、缺失了哪些
3. **定向改写建议**：哪些经历该上移、加重，措辞怎么贴近 JD 语言

比如 JD 写"高并发"，你简历里写的是"流量大"——它会帮你对齐用词。

**绝不编造经历和数字**，只在你真实具备的前提下调整表达和顺序。面试会被追问的东西，不能造假。

#### 5. 多格式输入，不用从零写

- **有表格**：CSV / xlsx 直接导入，自动转成简历结构
- **有旧简历**：Markdown 格式丢进去，在原有基础上润色
- **什么都没有**：对话式采集。AI 会按 STAR 思路追问——不只问"做了什么"，还会问"背景是什么、解决了什么问题、结果怎么量化"

不管哪种方式，最终都输出统一的 Markdown 内容文件，可以随时手动编辑，也可以用 git 做版本管理。

### 四、一个完整案例

来看一个从原始素材到最终 PDF 的完整流程：

**原始输入：**
> 我在沧澜智算实习，做后端开发，主要做了任务调度服务优化和灰度发布流程。

**AI 追问后挖掘出的细节：**
- 具体做了什么优化？→ 引入本地缓存与批量写入
- 效果呢？→ P99 延迟从 640ms 降到 210ms
- 规模多大？→ 日均 920 万次请求
- 灰度发布解决了什么问题？→ 故障恢复时间从 25 分钟缩短到 4 分钟
- 覆盖了多少服务？→ 覆盖团队 5 个核心服务

**润色后的简历条目：**
> - 优化任务调度服务，引入本地缓存与批量写入，将 P99 延迟从 640ms 降至 210ms，支撑日均 920 万次请求
> - 搭建灰度发布与自动回滚链路，将故障恢复时间从 25 分钟缩短到 4 分钟，覆盖团队 5 个核心服务

**选模板渲染：**
```
使用 $resume-builder，用 compact 模板、teal 配色渲染成 PDF
```

一份专业的 A4 简历就出来了。整个过程 5-10 分钟。

当前 compact 示例同时展示了头像排版；照片不是强制项，可以随时切回无头像版本。

### 五、和市面工具有什么不同

| 维度 | 在线简历网站 | 纯 AI prompt | resume-builder |
|------|-------------|-------------|----------------|
| 内容润色 | ❌ 自己写 | ✅ 有 | ✅ STAR 句式 + 量化追问 |
| 排版模板 | ✅ 有但要会员 | ❌ 纯文字 | ✅ 5 套内置模板 |
| PDF 输出 | ✅ 导出要钱 | ❌ 自己排 | ✅ 离线直接生成 |
| JD 匹配 | ❌ 无 | ⚠️ 看 prompt | ✅ 自动分析 |
| 中文优化 | ⚠️ 多数偏英文 | ⚠️ 看模型 | ✅ 专为中文求职设计 |
| 隐私 | ⚠️ 数据上传 | ⚠️ 看平台 | ✅ 全程本地 |
| 许可 | 💰 会员制 | 取决于模型平台 | 个人非商业使用免费，商用需授权 |

一句话总结：**别人帮你写内容，我帮你出成品。**

### 六、如何获取和使用

**安装到 Claude Code：**

```bash
git clone https://github.com/cosen1024/resume-builder-skill.git
cd resume-builder-skill
mkdir -p ~/.claude/skills
cp -R resume_skill ~/.claude/skills/resume-builder
```

**安装 Python 依赖（PDF 渲染需要）：**

```bash
python3 -m pip install weasyprint jinja2 python-frontmatter pyyaml openpyxl
```

**使用：**

```
使用 $resume-builder，根据我的真实经历生成一份后端开发校招简历。
使用 $resume-builder，润色这份简历，保留真实信息，不要编造数字。
使用 $resume-builder，根据这份 JD 调整我的简历，并列出命中和缺失的关键词。
使用 $resume-builder，把这份简历渲染成 compact 和 modern 两个版本。
```

也支持 Codex，安装方式见 GitHub README。
