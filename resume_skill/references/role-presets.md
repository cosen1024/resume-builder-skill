# 岗位内容预设（按岗位调整结构与侧重）

录入和「JD 定向润色」时参考本表，决定**小节顺序、强调什么、技能线怎么写**。
这是内容层指导，与视觉模板（classic/modern/timeline/minimal）正交——任意岗位都可搭配任意模板。

**通用规则**
- 有 JD 时，**JD 里反复出现的职责/技术词优先级最高**，本表只作兜底结构。
- 技能线挑 **4–6 个高信号项**，按岗位相关度排序，不要堆砌；没有真凭实据的不写。
- 校招把教育/竞赛靠前；社招把工作/项目成果靠前、教育下沉。
- 每条经历仍按 STAR/量化（见 `writing-principles.md`）。

## 技术研发类

| 岗位 | 推荐小节顺序（社招） | 强调 | 技能线示例 |
|------|---------------------|------|-----------|
| 后端 | 工作/项目经历 → 专业技能 → 教育 | 高并发、稳定性、数据层、可量化性能提升 | `Go/Java, PostgreSQL, Redis, Kafka, Docker/K8s` |
| 前端 | 工作/项目经历 → 专业技能 → 教育 | 工程化、性能/体验优化、组件化、跨端 | `React/Vue, TypeScript, Webpack/Vite, Node, 可视化` |
| 算法/AI | 项目/研究 → 论文竞赛 → 技能 → 教育 | 建模能力、指标提升、落地、论文/比赛 | `RAG/Agent, PyTorch, LangGraph, FastAPI, 向量库` |
| 客户端(iOS/Android) | 工作/项目 → 技能 → 教育 | 端性能、内存/包体优化、跨端、上架经验 | `Swift/Kotlin, 多线程, 网络层, 组件化, 监控` |
| 数据/大数据 | 工作/项目 → 技能 → 教育 | 数据管道、规模、查询/成本优化 | `Spark, Flink, Hive, SQL, 数仓建模` |
| 测试/QA | 工作/项目 → 技能 → 教育 | 自动化覆盖率、质量度量、CI、缺陷拦截 | `pytest/Selenium, 接口/性能测试, CI, 自动化框架` |
| 运维/SRE | 工作/项目 → 技能 → 教育 | 稳定性、SLA、自动化、故障处理时长 | `K8s, Prometheus, Ansible, CI/CD, Shell/Python` |
| 安全 | 工作/项目 → 技能/证书 → 教育 | 漏洞挖掘、攻防、合规、止损量化 | `渗透测试, WAF, 应急响应, 代码审计, 等保` |

## 产品 / 设计 / 分析类

| 岗位 | 推荐小节顺序 | 强调 | 关键内容 |
|------|------------|------|---------|
| 产品经理 | 工作/项目 → 技能 → 教育 | 业务结果（DAU/转化/营收）、需求与协作、数据驱动 | 负责的产品线、关键指标、从 0→1 或增长案例 |
| UI/UX 设计 | 作品/项目 → 技能 → 教育 | 设计成果、转化/体验提升、作品集链接 | 工具(Figma/Sketch)、设计系统、可量化效果；建议带证件照/作品集链接 |
| 数据分析 | 工作/项目 → 技能 → 教育 | 分析驱动决策、指标提升、建模 | `SQL, Python, Tableau/Power BI, A/B 实验, 统计建模` |

## 职能 / 运营类

| 岗位 | 推荐小节顺序 | 强调 | 关键内容 |
|------|------------|------|---------|
| 市场/品牌 | 工作/项目 → 技能 → 教育 | 投放 ROI、增长、活动效果、预算规模 | 渠道、转化漏斗、可量化战绩 |
| 用户运营 | 工作/项目 → 技能 → 教育 | 留存/活跃/增长、社群规模、运营策略 | 拉新/促活/留存数据、活动复盘 |
| HR/HRBP | 工作经历 → 技能/证书 → 教育 | 招聘效率、组织/绩效、覆盖人数 | 招聘漏斗、人效、制度落地 |
| 项目经理(PMP) | 工作经历 → 证书 → 教育 | 交付准时率、预算、跨团队协调、范围 | 项目规模、里程碑、风险管控；突出 PMP 证书 |

## 应届/校招微调
- 教育背景置顶，写清学校/专业/GPA/排名。
- 用**竞赛、课程项目、实习、社团**补足经历厚度；技能可列得更全以示学习力。
- 没有工作经历时，「项目经验」是主战场，按 STAR 深写 1–2 个。
