# resume.md 数据格式约定

`resume.md` 既是人类可读可手改的简历，又能被 `scripts/render.py` 解析后套用模板。
分两部分：**YAML front matter**（原子字段）+ **Markdown 正文**（分节经历）。

## YAML front matter

放姓名、联系方式、求职意向、个人简介等。改一处全局生效。

```yaml
---
name: 库森                 # 必填
gender: 男                 # 可选
birth: 1999-11            # 可选
phone: 18826754310
email: kusen.dev@example.com
location: 杭州市西湖区
hometown: 浙江省           # 可选
intent: 后端开发工程师      # 求职意向（标题），用于头部展示
intent_detail: 期望地点：杭州 · 到岗时间：2026.06   # 可选，附加信息
summary: >-                # 个人简介/自我评价，校招可留、社招多删
  一句到三句话的概述……
# photo: ./avatar.jpg     # 可选证件照，路径相对 resume.md；不写则不显示（投外企建议不放）
# id_number: 3301...      # 隐私字段，默认注释，不输出
---
```

渲染时各字段的用法：classic/timeline/minimal 头部显示 `name + intent + 联系方式 + summary`；
modern 把它们放进左侧边栏。`photo` 由 compact/classic/modern/minimal 渲染，
timeline 为保持时间线结构不显示照片。配色不写在这里，用渲染命令的
`--accent`（见 `templates-guide.md`）。`render.py` 容错——缺哪个字段就跳过该处。

## 正文：用 `## ` 分节

每个 `## ` 是一个小节。节标题建议用标准名（影响 ATS 与 modern 的分栏，见下）：
`教育背景` `实习经历` `工作经历` `项目经验` `校园经历` `专业技能` `荣誉奖项`。

### 经历型小节：用 `### ` 表示一条条目

```markdown
## 实习经历

### 沧澜智算 · 基础架构部 · 后端开发实习生 | 2025.08 – 2025.12 · 南京
- 优化任务调度服务，将 P99 延迟从 640ms 降到 210ms，支撑日均 920 万次请求
- 搭建灰度发布与自动回滚链路，将故障恢复时间从 25 分钟缩短到 4 分钟
```

解析规则（`render.py` 的 `parse_entry_heading`）：
- `### ` 行用 ` | ` 分成**左半（标题）**与**右半（时间/地点等元信息）**。
- 左半用 ` · ` 分隔成多段：**第 1 段加粗**（公司/学校/项目名），其余为副标题（部门/职位/角色）。
- 右半整体作为右对齐的灰色元信息（时间、地点）。
- 之后 `- ` 开头的行是该条目的要点（支持 `**加粗**`、`*斜体*`、`` `代码` ``）。

### 列表型小节：直接用 `- `，不写 `### `

适合技能、荣誉这类无需"条目标题"的内容：

```markdown
## 专业技能
- **编程语言**：Go、Python、Java
- **框架/中间件**：Kubernetes、Redis、Kafka

## 荣誉奖项
- 2025.05 · 蓝桥杯全国软件和信息技术专业人才大赛 · 全国一等奖
```

## modern 模板的左栏分配

`render.py` 按节标题关键词把小节分到**左侧边栏**还是**主栏**：
标题含 `技能 / 证书 / 求职 / 意向 / 语言 / 联系` 之一 → 进侧边栏；其余进主栏。
（关键词表见 `render.py` 的 `SIDEBAR_KEYWORDS`，可按需扩展。）
所以「专业技能」「语言能力」这类会自动落到左栏，「教育背景」「实习经历」「项目经验」落到主栏。

## 从 CSV/xlsx 导入

`scripts/csv_to_md.py` 把「分类,字段名,值」三列表格转成上面的 `resume.md`。
可直接参考 `assets/resume.example.csv`：
- `基本信息` → front matter；`求职意向` → intent/intent_detail；`自我评价` → summary。
- `教育背景/本科教育` → 教育背景节；`实习经历N/工作经历N` → 实习/工作经历节（N 自动合并）。
- `项目经验/校园项目` → 项目经验；`校园活动` → 校园经历；`技能证书` → 专业技能；`获奖情况` → 荣誉奖项。
- 值里的 `\n` 或换行、以及 `1. / 2、` 这类序号会被拆成独立要点。
- 支持 `.csv` 和 `.xlsx`；旧版 `.xls` 请先转换格式。
- 导入结果是**骨架**：务必再按 `writing-principles.md` 把要点改写为 STAR/量化，再渲染。
